This document describes the complete behavioral contract for the **Task** resource. It is intended for product owners, policy staff, and other non-technical reviewers.

## How to read this document

- **States** ÔÇö the lifecycle stages each resource can be in, and how each affects SLA tracking.
- **Triggers** ÔÇö automatic reactions to lifecycle events: object creation, field changes, external events, and timers (onCreate, onUpdate, onEvent, onTimer).
- **Operations** ÔÇö actor- or system-triggered actions that can move a resource to a new state or update it in place.
- **Guards** ÔÇö named conditions that control who can trigger an operation. If a guard fails, the request is rejected.
- **Rules** ÔÇö named procedures that handle logic spanning multiple resources or that apply in several places.

To propose a change, there are two paths:
- **Non-technical:** Edit the CSV files in this folder and ask a developer to run `npm run contract-tables:import` to apply your changes to the source YAML.
- **Technical:** Edit the source YAML files directly and submit a pull request.

## States

The **SLA clock** tracks time toward resolution:
- **Running** ÔÇö time is counting toward the SLA deadline
- **Paused** ÔÇö time is not counting (resource is blocked, waiting on external input)
- **Stopped** ÔÇö work is complete; SLA is no longer tracked

## Triggers

Triggers are automatic reactions to lifecycle events. Each lists what fires it and what steps run.

### Task

**On create** ÔÇö runs when a new record is created:

- Route task to a queue based on subject and program
- Set task priority on creation

**On update** ÔÇö runs when any of the following fields change: `isExpedited`, `programType`, `queueId`:

- Re-evaluate priority when expedited status or program type changes

## Guards

Guards are conditions checked before a transition fires. A transition will not execute unless all of its guards pass. Multiple guards on a transition use AND logic; `any of:` within a guard uses OR logic.

| Guard | Condition |
|-------|-----------|
| `taskIsUnassigned` | `assignedToId` is not set |
| `callerIsAssignedWorker` | `assignedToId` = `$caller.id` |
| `callerIsSupervisor` | `$caller.roles` contains any of `supervisor` |
| `callerIsSystem` | `$caller.roles` contains any of `system` |
